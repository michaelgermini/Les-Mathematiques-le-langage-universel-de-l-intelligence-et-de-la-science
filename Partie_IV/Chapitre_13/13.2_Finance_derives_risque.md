# 13.2. Finance, dérivés et risque

## Introduction

La finance mathématique applique les outils du calcul stochastique et de l'optimisation pour évaluer les actifs financiers, gérer les risques et construire des portefeuilles. Ce chapitre présente les modèles fondamentaux, du CAPM à Black-Scholes, et les techniques modernes de gestion du risque.

---

## 1. Théorie du portefeuille

### 1.1 Modèle de Markowitz

#### Cadre
- n actifs avec rendements R = (R₁, ..., Rₙ)
- Espérance : μ = E[R]
- Covariance : Σ = Cov(R)

#### Problème d'optimisation
```
min w'Σw    (variance du portefeuille)
sous : w'μ = μ_cible    (rendement cible)
       w'1 = 1          (somme des poids = 1)
```

#### Solution
```
w* = Σ⁻¹(λ₁μ + λ₂1)
```
où λ₁, λ₂ sont les multiplicateurs de Lagrange.

#### Frontière efficiente
Ensemble des portefeuilles de variance minimale pour chaque niveau de rendement.

```
Rendement
    │     ●●●●●
    │   ●●    ●●
    │  ●        ●
    │ ●          ●  ← Frontière efficiente
    │●
    └────────────────→ Risque (σ)
```

### 1.2 Modèle CAPM

#### Capital Asset Pricing Model
```
E[Rᵢ] = Rf + βᵢ(E[Rm] - Rf)
```

où :
- Rf : taux sans risque
- Rm : rendement du marché
- βᵢ = Cov(Rᵢ, Rm)/Var(Rm) : sensibilité au marché

#### Interprétation
- Le seul risque rémunéré est le risque systématique (β)
- Le risque spécifique peut être diversifié

#### Security Market Line (SML)
```
E[R]
  │
  │        ●  Actif surévalué (sous la ligne)
  │      /
  │    /●  Actif correctement évalué
  │  /
  │/●  Actif sous-évalué (au-dessus de la ligne)
  Rf
  └────────────────→ β
```

### 1.3 Modèles multi-facteurs

#### Fama-French (3 facteurs)
```
E[Rᵢ] - Rf = βᵢ,MKT(Rm - Rf) + βᵢ,SMB·SMB + βᵢ,HML·HML
```
- **SMB** : Small Minus Big (taille)
- **HML** : High Minus Low (value)

#### Extensions
- Momentum (Carhart, 4 facteurs)
- Profitabilité, investissement (Fama-French, 5 facteurs)
- Facteurs de risque alternatifs

## 2. Évaluation des dérivés

### 2.1 Options : définitions

| Type | Payoff à maturité |
|------|-------------------|
| **Call européen** | max(S_T - K, 0) |
| **Put européen** | max(K - S_T, 0) |
| **Call américain** | Exercice possible avant T |
| **Put américain** | Exercice possible avant T |

où S_T est le prix du sous-jacent à maturité et K le strike.

### 2.2 Modèle de Black-Scholes

#### Hypothèses
- Le prix S suit un mouvement brownien géométrique :
```
dS = μS dt + σS dW
```
- Pas d'arbitrage, marché complet
- Taux sans risque r constant

#### Équation de Black-Scholes
```
∂V/∂t + ½σ²S²∂²V/∂S² + rS∂V/∂S - rV = 0
```

#### Formule pour un call européen
```
C = S·N(d₁) - K·e^{-rT}·N(d₂)
```
où :
```
d₁ = [ln(S/K) + (r + σ²/2)T] / (σ√T)
d₂ = d₁ - σ√T
```
N(·) est la fonction de répartition normale.

#### Formule pour un put européen (parité put-call)
```
P = K·e^{-rT}·N(-d₂) - S·N(-d₁)
```

### 2.3 Les Grecs

Sensibilités du prix de l'option aux paramètres :

| Grec | Définition | Interprétation |
|------|------------|----------------|
| **Delta (Δ)** | ∂V/∂S | Sensibilité au sous-jacent |
| **Gamma (Γ)** | ∂²V/∂S² | Convexité |
| **Theta (Θ)** | ∂V/∂t | Décroissance temporelle |
| **Vega (ν)** | ∂V/∂σ | Sensibilité à la volatilité |
| **Rho (ρ)** | ∂V/∂r | Sensibilité au taux |

#### Couverture Delta
Maintenir un portefeuille Δ-neutre :
```
Π = V - Δ·S
dΠ = 0 (au premier ordre)
```

### 2.4 Au-delà de Black-Scholes

#### Volatilité implicite
La volatilité σ qui égalise le prix de marché et le prix BS.

#### Smile de volatilité
La volatilité implicite varie avec le strike :
```
σ_impl
    │  ╲      ╱
    │   ╲    ╱
    │    ╲__╱
    │
    └─────────────→ K/S
        ATM
```

#### Modèles avancés
- **Volatilité locale** : σ(S, t)
- **Volatilité stochastique** : Heston, SABR
- **Sauts** : Merton jump-diffusion
- **Modèles à volatilité rugueuse** : rough volatility

## 3. Gestion du risque

### 3.1 Value at Risk (VaR)

#### Définition
Le VaR au niveau α est la perte maximale avec probabilité 1-α :
```
P(Perte > VaR_α) = α
```

#### Exemple
VaR 99% sur 1 jour = 1 million € signifie :
"Il y a 1% de chances de perdre plus d'1 million € en un jour."

#### Méthodes de calcul

| Méthode | Principe | Avantages/Inconvénients |
|---------|----------|-------------------------|
| **Historique** | Quantile empirique | Simple, pas d'hypothèse de distribution |
| **Paramétrique** | Hypothèse normale | Rapide, mais queues fines |
| **Monte Carlo** | Simulation | Flexible, mais coûteux |

#### Formule paramétrique (normale)
```
VaR_α = μ - σ·z_α
```
où z_α est le quantile de la loi normale (z_0.01 ≈ 2.33).

### 3.2 Expected Shortfall (ES)

#### Définition
Perte moyenne conditionnelle au-delà du VaR :
```
ES_α = E[Perte | Perte > VaR_α]
```

#### Avantages sur le VaR
- **Cohérent** : satisfait la sous-additivité
- **Informative** : capture la sévérité des pertes extrêmes

### 3.3 Stress testing

#### Scénarios historiques
- Crise de 2008
- Krach de 1987
- Crise COVID 2020

#### Scénarios hypothétiques
- Hausse des taux de 200 bp
- Chute des marchés de 30%
- Défaut d'une contrepartie majeure

### 3.4 Risque de crédit

#### Modèle de Merton
La dette d'une entreprise est vue comme une option :
- Les actionnaires détiennent un call sur les actifs
- Les créanciers ont vendu un put

#### Probabilité de défaut
```
PD = N(-d₂)
```
où d₂ est calculé avec les actifs de l'entreprise.

#### Credit Default Swaps (CDS)
- L'acheteur paie une prime périodique
- Le vendeur compense en cas de défaut
- Le spread CDS reflète le risque de crédit

## 4. Produits dérivés avancés

### 4.1 Swaps

#### Interest Rate Swap
Échange de flux :
- Payeur fixe : paie taux fixe, reçoit taux variable
- Payeur variable : inverse

#### Valorisation
Valeur = PV(flux reçus) - PV(flux payés)

### 4.2 Options exotiques

| Type | Caractéristique |
|------|-----------------|
| **Barrière** | Activation/désactivation si S franchit un seuil |
| **Asiatique** | Payoff sur moyenne de S |
| **Lookback** | Payoff sur max/min de S |
| **Digitale** | Payoff binaire (0 ou fixe) |
| **Quanto** | Sous-jacent en devise étrangère, payoff en devise locale |

### 4.3 Produits structurés

Combinaisons d'obligations et de dérivés :
- Capital garanti + participation à la hausse
- Autocall : remboursement anticipé si conditions remplies
- Reverse convertible : rendement élevé mais risque de conversion en actions

## 5. Modèles de taux d'intérêt

### 5.1 Courbe des taux

#### Structure par terme
```
Taux
  │      ____
  │    /
  │  /
  │/
  └────────────→ Maturité
```
Normale (croissante), inversée, plate, bossue.

#### Taux zéro-coupon
Prix d'une obligation zéro-coupon :
```
P(t, T) = e^{-r(t,T)(T-t)}
```

### 5.2 Modèles de taux courts

#### Vasicek
```
dr = a(b - r)dt + σdW
```
Retour à la moyenne, mais taux négatifs possibles.

#### Cox-Ingersoll-Ross (CIR)
```
dr = a(b - r)dt + σ√r dW
```
Taux toujours positifs.

#### Hull-White
```
dr = (θ(t) - ar)dt + σdW
```
Calibration exacte à la courbe observée.

### 5.3 Modèles HJM et de marché

#### Heath-Jarrow-Morton
Modélise directement les taux forward :
```
df(t,T) = α(t,T)dt + σ(t,T)dW
```

#### Modèle de marché (BGM/LMM)
Modélise les taux LIBOR forward.
Standard pour les produits de taux complexes.

## 6. Considérations pratiques

### 6.1 Calibration

#### Objectif
Ajuster les paramètres du modèle aux prix de marché observés.

#### Méthodes
- Minimisation des erreurs quadratiques
- Maximum de vraisemblance
- Méthode des moments

### 6.2 Risque de modèle

- Les modèles sont des approximations
- Importance des tests de robustesse
- Réserves pour incertitude de modèle

### 6.3 Régulation

- **Bâle III** : exigences en capital pour les banques
- **Solvabilité II** : pour les assureurs
- **EMIR** : compensation centrale des dérivés

---

## Conclusion

La finance mathématique fournit les outils pour :

1. **Évaluer** les actifs financiers de manière cohérente (absence d'arbitrage)
2. **Construire** des portefeuilles optimaux (Markowitz, CAPM)
3. **Mesurer** et **gérer** les risques (VaR, ES, couverture)
4. **Innover** avec de nouveaux produits (dérivés exotiques, structurés)

Du calcul stochastique aux méthodes numériques, les mathématiques sont au cœur de la finance moderne.
